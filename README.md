## 📂 How to Navigate This Repo

If you are here to study along, follow this order:

1. Start with `30-Day-Challenge/` — go day by day, do not skip ahead.
2. After finishing the 30 days, move to `Topic-Wise/` for deeper practice per topic.
3. Each `notes.md` explains the pattern in plain language — read it before attempting the day's problems.
4. Solutions are in individual `.py` files, named by problem. Try the problem yourself first before reading the solution.

## 💡 My Approach Philosophy

I follow three rules in this repo:

**1. Pattern before code.**  
Before writing any solution, I identify which pattern applies — two pointer, sliding window, prefix sum, etc. — and write it down. This is the actual skill interviews test.

**2. Struggle time is learning time.**  
I give every problem a genuine 20–25 minutes of solo attempt before consulting any hint or editorial. Skipping this step makes solutions feel recognizable without being usable.

**3. No copy-paste solutions.**  
Every solution here is typed by hand and commented in my own words. The goal is understanding, not completion.
